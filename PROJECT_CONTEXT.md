# PROJECT CONTEXT — STM32 计数器项目

## 项目概述

基于 STM32F103 的计数/定长控制系统。传感器脉冲计数，达到目标值后电机自动停止，冷却后自动重启，循环运行。

## 状态机

```
         ┌──────────────────────────────┐
         │   CONFIRM / RESET            │
         │   (清零, 停电机)              │
         ▼                              │
      ┌──────┐  CONFIRM(target>0)  ┌──────────┐
      │ IDLE │ ──────────────────→ │ RUNNING  │
      │      │                     │ 电机 ON   │
      └──────┘                     └──────────┘
         ▲                              │
         │                              │ current ≥ target
         │                              ▼
         │                        ┌───────────┐
         │   CONFIRM / RESET      │ COOLDOWN  │
         └─────────────────────── │ 电机 OFF   │
                                  │ LED10 ON  │
                                  └───────────┘
                                       │
                                       │ 5s 后
                                       │ 自动重启电机
                                       ▼
                                  ┌──────────┐
                                  │ RUNNING  │  (下一轮)
                                  └──────────┘
```

- **IDLE**: PB1(ADD) 调整 target_count (0~99 循环)，PC13 短按启动 (target>0 时)，PC13 长按清零
- **RUNNING**: 电机 60% 占空比运行，CountSensor delta 累加 current_count（app 层 120ms 防抖），每有效脉冲 PA9 闪 50ms，达标→COOLDOWN。CONFIRM/RESET 清零回 IDLE
- **COOLDOWN**: 电机停止，LED10 亮，等 5s → 自动清零 current_count 回 RUNNING。CONFIRM/RESET 清零回 IDLE

## 已知问题：电机干扰

### 现象

- 接入电机电源后，电机启动/运行会严重干扰 PB14 红外计数输入
- 串口日志出现 `IR raw=25`、`IR raw=30` 等异常大脉冲
- 可能导致 STM32 异常复位或主循环卡死（OLED 熄屏、系统停止响应）
- 断开电机电源后，仅保留 STM32、OLED、红外、按键、USB-TTL，计数恢复正常（raw=1）

### 根因分析

电机 PWM 驱动产生电磁干扰（EMI），通过以下路径影响 PB14：

1. **传导干扰**：电机和 STM32 共用电源，电机启停产生电压波动
2. **辐射干扰**：电机换向火花/高频 PWM 通过电磁场耦合到 PB14 信号线
3. **地弹**：电机大电流在地线上产生电压差，干扰红外传感器参考电平

当前的 app 层 120ms 红外防抖只能过滤计数增量，无法阻止 ISR 层面的中断风暴。

### 临时处理

- 后续软件开发和逻辑测试时**先断开电机电源**
- 只保留 STM32、OLED、红外、按键、USB-TTL 测试主逻辑
- 电机相关逻辑可正常开发，但验证时以无电机环境为准

### 后续硬件处理计划

1. **电机单独供电** — 电机驱动板使用独立电源，不与 STM32 共用
2. **共地** — STM32 与电机驱动板 GND 相连（仅地线，电源隔离）
3. **电机驱动 VM/GND 并联电解电容** — 100μF~470μF，吸收电机启停电流尖峰
4. **电机两端并联陶瓷电容** — 0.1μF (104)，滤除换向火花高频噪声
5. **信号线远离电机线** — PB14 红外信号线远离电机驱动线和电机本体
6. **PB14 输入端 RC 滤波** — 串联 1kΩ 电阻 + 对地 100nF 电容 + 10kΩ 上拉
7. **软件兜底** — app 层已加入 `APP_IR_DEBOUNCE_MS=120ms` 最小计数间隔过滤

## 当前限制

1. **CountSensor 需要外部脉冲源** — PB14 无脉冲时计数不增长
2. **App_Task 中 GetEvent 在 Update 之前调用** — 按键事件有 1 帧延迟（非致命）
3. **LED ON/OFF 函数每次重新初始化 GPIO** — 功能正确但低效
4. **system/ 和部分 hardware/ 旧驱动未清理** — 可编译但未使用
5. **电机干扰未硬件解决** — 当前接电机时系统不稳定，需断开电机测试
6. **无 printf 重定向** — 日志使用逐字符阻塞发送，波特率 115200
7. **无 USART 中断/DMA** — TX 为轮询等待 TXE 标志位

## 红外计数与 LED 调试记录

**日期**：2026-06-08

### 背景

PCB V1 逻辑部分核心链路（STM32 + OLED + 按键 + 状态机）验证通过后，进入红外计数调试阶段。此阶段还将原 3Pin 红外传感器更换为 4Pin 模块（AO/DO/GND/VCC），使用 DO 数字输出接 PB14。

### 最初异常现象 🔴

| 现象 | 表现 |
|------|------|
| 传感器本体电压变化 | 万用表测传感器 OUT 端，遮挡/未遮挡时电压有变化 |
| OLED 计数不增加 | 反复遮挡传感器，OLED 上 C 值始终为 0 |
| 串口日志 PB14 lvl 恒为 1 | `PB14 lvl=1` 持续输出，无论遮挡与否都不变 |

### 排查过程

**第一步（误判）**：怀疑传感器输出电平异常。更换为 4Pin 模块后问题依旧，排除传感器本身故障。

**第二步（关键测量）**：分别测量 H4 OUT、R1 两端对 GND 电压：

| 测量点 | 未遮挡 | 遮挡 | 说明 |
|--------|--------|------|------|
| 传感器 OUT（H4 输入端） | ~0V | ~3V | ✅ 传感器输出正常，可稳定翻转 |
| R1 靠传感器侧 | ~0V | ~3V | ✅ 信号经过 R1 前正常 |
| R1 靠 PB14 侧（IR 测试点） | ~1.6V | ~3V | ❌ 低电平被抬高！ |

**第三步（定位根因）**：R1 后端 / PB14 侧低电平被抬高的原因是 **R2 上拉配置不合适**，导致 PB14 低电平从 0V 被拉到约 1.6V 中间电压。STM32F103 的 V_IH(min) = 2.145V、V_IL(max) = 1.155V，1.6V 处于不确定区域，STM32 实际读取为高电平，因此 `PB14 lvl` 始终为 1，上升沿中断永不触发。

### 临时修复方案

| 元件 | 处理 | 说明 |
|------|------|------|
| R2 | **暂时空着（不焊）** | 移除不当上拉，消除分压效应 |
| R1 | 保留当前临时配置 | 串联限流，保护 PB14 |
| C6 | 保留 | 去耦/滤波，不影响数字信号 |

修复后测量结果：

| 测量点 | 未遮挡 | 遮挡 |
|--------|--------|------|
| PB14 / IR 测试点 | ~0V | ~3V |
| 串口 `PB14 lvl` | `0` | `1` |

电平翻转干净、稳定，STM32 可正确识别。

### 软件侧配合

- 触发沿保持 **上升沿** `EXTI_Trigger_Rising`（未遮挡=0 → 遮挡=1 上升沿触发计数）
- `CountSensor_ReadRawLevel()` 和周期性 `PB14 lvl` 串口日志保留，用于持续监控
- 当前代码不需任何修改，红外触发后 OLED 计数已恢复正常

### LED 指示

之前 0805 LED 及相关贴片元件未就绪的问题也已解决，当前 **LED9（计数闪烁）和 LED10（COOLDOWN 常亮）均可正常工作**，与红外计数和状态机联动正确。
