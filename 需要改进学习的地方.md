## 2. 非阻塞思想

这是你这次项目里最核心的进步。

你原 51 项目最大坑是：

```c
while(IR_SENSOR == 0);
Delay_ms(5000);
```

这种写法会卡住 CPU，导致：

```text
OLED/数码管刷新变慢
按键没响应
电机状态不及时
系统像“死机”
```

现在 STM32 版改成了：

```c
if (App_Millis() - cooldown_start_ms >= 5000)
{
    // 时间到了再执行
}
```

你必须吃透这句话：

> 嵌入式系统里，主循环不能被一个任务长时间霸占。

你要能讲清楚：

```text
为什么不用 Delay_ms(5000)？
为什么按键消抖不用 Delay_ms(20)？
为什么 LED 闪 50ms 也不用 Delay_ms(50)？
为什么 OLED 不每轮都刷新？
```

这个是你这个项目真正的技术亮点。

## 3. 状态机

你这次项目的灵魂就是状态机：

```text

COOLDOWN：达标停机，等待 5 秒

COOLDOWN 结束为什么不是回 IDLE，而是回 RUNNING？
```

这是面试时最适合讲的部分。

## 3. ST-LINK 和 USB-TTL 的区别

你也反复问串口接线。

记住：

```text
ST-LINK：烧录程序、调试程序
USB-TTL / CH340：看串口日志
```

串口日志不是从 ST-LINK 看，是从 USB-TTL 看。

USART3 接线：

```text
STM32 PB10 / USART3_TX → USB-TTL RXD
STM32 GND              → USB-TTL GND
```

不要接反，不要接 5V。

串口一定是：

```text
TX → RX
RX → TX
GND → GND
```

现在你只发日志，所以只接：

```text
PB10 → RXD
GND  → GND
```
