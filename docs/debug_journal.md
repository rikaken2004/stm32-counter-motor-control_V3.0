# 调试问题日志

> 以下为项目推进过程中遇到的真实问题，按发现时间顺序记录。每个问题按"现象 → 初步判断 → 排查过程 → 最终解决 → 收获"格式组织。

---

## 问题 1：Arduino App Lab 显示 No boards found

- **现象**：插上 ESP32-S3 后，软件显示 "No boards found"，无法选择开发板
- **初步判断**：以为是驱动或 USB 线问题
- **排查过程**：重新插拔 USB，换线，仍不行。后来注意到桌面图标名是 "Arduino App Lab"
- **最终解决**：应使用 **Arduino IDE**（不是 App Lab），下载安装正确的 IDE 后正常识别
- **收获**：软件名差一个词可能就是完全不同的工具。嵌入式开发前先确认使用正确的 IDE

---

## 问题 2：Boards Manager 和 Library Manager 混淆

- **现象**：在 Library Manager 里搜索 `esp32`，找不到开发板
- **初步判断**：以为需要先安装库
- **排查过程**：看了教程才意识到要安装的是开发板包（Boards Manager），不是库
- **最终解决**：在 **Boards Manager** 搜索 `esp32 by Espressif Systems` 安装
- **收获**：Boards Manager 装的是平台支持包（编译器+SDK），Library Manager 装的是第三方代码库。两个入口不能混淆

---

## 问题 3：ESP32 双 Type-C 接口选择

- **现象**：插了 ESP32-S3 的一个 Type-C 口，电脑没反应
- **初步判断**：以为开发板坏了
- **排查过程**：ESP32-S3 开发板有两个 Type-C 口：一个走 CH340（USB-UART），一个走原生 USB（JTAG/CDC）
- **最终解决**：优先使用显示 **CH340** 的那个口（通常标注为 USB-UART 或 COM）。烧录需要用这个口
- **收获**：双 USB 开发板要先确认哪个口是烧录用

---

## 问题 4：多个 GND 是否都要接

- **现象**：看到开发板有好几个 GND 引脚，不确定是否每个都要连
- **初步判断**：不清楚
- **排查过程**：用万用表通断档测试，发现同一个开发板上所有 GND 都是内部互通的
- **最终解决**：只需接**一个 GND** 到公共地即可
- **收获**：同一块板子上的 GND 引脚是连通的，不需全接。但不同板子之间必须共地

---

## 问题 5：是否要连接 3.3V

- **现象**：两块开发板互相通信，不确定是否要互相提供 3.3V
- **初步判断**：不确定
- **排查过程**：查阅资料确认
- **最终解决**：两块板**各自 USB 供电**时，**不要连接 3.3V**。只需共地 (GND ↔ GND)
- **收获**：两个独立供电的设备通信只需共地和信号线，不需要电源互连。连接 3.3V 可能造成电源冲突损坏板子

---

## 问题 6：STM32 → ESP32 收到乱码

- **现象**：ESP32 串口监视器能收到数据，但显示乱码（不可读字符）
- **初步判断**：波特率不对
- **排查过程**：按顺序排查：①ESP32 自身 Serial.print 正常 → 电脑串口速率没问题；②STM32 接 CH340 在 115200 下正常 → STM32 输出没问题；③检查接线 → 发现 GND 未共地
- **最终解决**：接好 GND 后乱码消失。后来也遇到过 TX/RX 接反导致收不到任何数据的情况
- **收获**：串口乱码四步排查法：波特率 → GND → TX/RX 方向 → GPIO 选择

---

## 问题 7：网页有 Raw 数据但状态栏为 `--`

- **现象**：浏览器打开 ESP32 网页，Raw UART Data 区域有数据（如 `KEY ADD target=3`），但 STATE/TARGET 等卡片显示为 `--`
- **初步判断**：以为是解析函数 `getValue()` 有问题
- **排查过程**：检查 `updateData()` 逻辑。发现 STM32 不只输出 `STATE=...`，还会输出 `KEY ADD target=3`、`MOTOR RUN START speed=35`、`CMD RESET OK` 等日志行。这些行不含 `STATE=`，`getValue()` 找不到对应 key，返回 `--`，把之前正确的状态值覆盖了
- **最终解决**：在 `updateData()` 开头增加 `if (!line.startsWith("STATE=")) return;`，只解析状态行，其他日志只显示在 Raw UART Data 区域
- **收获**：数据解析要加过滤。上游可能输出多种格式，不能假设每一行都是你要解析的格式

---

## 问题 8：点击 RESET 后如何判断成功

- **现象**：点击网页 RESET 按钮后，不确定 STM32 是否真的复位了
- **初步判断**：需要多种方式交叉验证
- **排查过程**：设计了三种同时验证：
  1. 看 OLED → 是否显示 `State:IDLE`，`T:00 C:00`
  2. 看 STM32 串口助手 → 是否有 `CMD RESET OK`
  3. 看网页 Raw UART Data → 是否显示 `CMD RESET OK`
- **最终解决**：三种验证同时通过，确认 RESET 命令生效
- **收获**：验证远程控制效果时要多维度确认。不能只看网页（可能缓存），要看实际物理设备
