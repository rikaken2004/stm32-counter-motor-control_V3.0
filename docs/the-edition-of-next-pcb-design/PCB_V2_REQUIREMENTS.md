# STM32-ESP32 HIL 控制器测试板 V2 需求规格

> 规划文档。目标：冻结 V2 PCB 功能边界和接口清单，不画 PCB，不生成 Gerber。

## 1. V2 PCB 定位

V2 PCB 不定位为完整无人机飞控板。

V2 PCB 定位为：

**STM32-ESP32 最小 HIL 控制器测试板 V2**

核心目标：

- 固化当前 STM32 + ESP32 HIL 接线（GPIO4 → PB14 虚拟脉冲注入、USART3 状态反馈）
- 提高测试稳定性，减少杜邦线接触不良
- 引出清晰的测试点，方便逻辑分析仪夹取
- 支持虚拟传感器输入注入
- 支持 STM32 状态反馈
- 支持后续 IMU / 舵机 / 多接口扩展
- 为飞控控制器测试、HIL 半实物仿真和硬件接口联调做准备

## 2. 为什么不直接画完整飞控板

| 原因 | 说明 |
|------|------|
| 验证范围 | 当前项目主要验证 HIL 输入注入和状态反馈，不是真实飞控算法 |
| 外设未冻结 | MPU6050、舵机、RS485、CAN 等外设尚未全部验证 |
| 接口未冻结 | 电源、电机驱动、执行器接口尚未最终确定 |
| 复杂度 | 完整飞控板涉及传感器布局、振动、电源噪声、EMI、执行器安全等 |
| 返工风险 | 直接画大板返工风险高、成本高、调试周期长 |
| 迭代策略 | 先做低风险、低成本、功能明确的 HIL 控制器测试载板，验证通过后再规划完整飞控板 |

## 3. 与公司产品方向的对应关系

| 公司产品方向 | V2 PCB 对应设计 |
|-------------|----------------|
| 飞控控制器 | STM32 控制器 + 传感器/执行器扩展接口 |
| HIL 半实物仿真 | ESP32 虚拟传感器输入注入（GPIO4 → PB14） |
| 桌面仿真设备 | GPIO 脉冲注入 + USART 状态反馈 + 测试点 |
| 冗余 IMU / 气压计 / 磁罗盘 | 双 I2C 传感器扩展接口 |
| 实时系统 / 状态反馈 | USART3 状态输出 `STATE=...`，ESP32 读取 |
| 多接口通信 | UART / RS485 / CAN 预留 |
| 信号级测试 | PB14、GPIO4、USART、PWM 等测试点引出 |
| 数据记录与分析 | Linux / PC 串口日志记录接口（CH340 / USB-TTL） |

## 4. V2 PCB 必做模块

### 4.1 HIL 虚拟传感器注入模块

核心功能：ESP32 GPIO4 输出虚拟传感器脉冲 → 注入 STM32 PB14。

设计要点：

- ESP32 GPIO4 → 1kΩ 串联电阻 → 3Pin 跳线选择 → STM32 PB14
- 跳线三选一：真实传感器 / ESP32 虚拟输入 / 断开（高阻）
- 跳线避免真实传感器和 ESP32 同时对 PB14 强驱动

当前已通过飞线验证。V2 将其固化到 PCB 走线。

### 4.2 STM32 与 ESP32 状态反馈串口模块

核心功能：STM32 USART3 状态输出 → ESP32 UART1 读取；ESP32 TX → STM32 RX 命令下发。

- STM32 PB10 (USART3_TX) → ESP32 GPIO18 (UART1 RX)
- STM32 PB11 (USART3_RX) ← ESP32 GPIO17 (UART1 TX)：预留，已通过 RESET 命令验证
- 留出 4Pin 排针（GND / STM32_TX / STM32_RX / 3.3V）供逻辑分析仪或 CH340 接入

### 4.3 逻辑分析仪测试排针

至少引出以下信号，排成 1×N 或 2×N 排针，方便逻辑分析仪夹取：

| 序号 | 信号 | 说明 |
|------|------|------|
| 1 | GND | 逻辑分析仪公共地 |
| 2 | ESP32 GPIO4 | 虚拟传感器脉冲输出 |
| 3 | STM32 PB14 | 计数输入（注入点后端） |
| 4 | STM32 USART3_TX (PB10) | 状态输出 |
| 5 | STM32 USART3_RX (PB11) | 命令接收 |
| 6 | PWM (PA6) | 电机控制信号 |
| 7 | 3.3V | 参考电平 |

所有测试点应为通孔焊盘或测试环，间距兼容 2.54mm 排针。

### 4.4 电源与基础保护

- 5V 输入（Micro USB 或接线端子）
- 3.3V 测试点（通孔焊盘 + 丝印标注）
- GND 测试点（至少 2 个）
- 电源 LED（3.3V 指示灯）
- 复位按钮（NRST）
- BOOT0 选择（跳线或按钮，用于 STM32 下载模式）
- 关键 IO 串联保护电阻（PB14、GPIO4）

### 4.5 SWD 下载调试接口

标准 4Pin/5Pin 排针：

| 引脚 | 信号 |
|------|------|
| 1 | 3.3V |
| 2 | SWDIO |
| 3 | SWCLK |
| 4 | GND |
| 5 (可选) | NRST |

兼容 ST-Link V2 的 SWD 接口。

## 5. V2 PCB 强烈建议模块

### 5.1 I2C IMU1 接口

| 引脚 | 信号 |
|------|------|
| 3.3V | 电源 |
| GND | 地 |
| SCL | I2C 时钟 |
| SDA | I2C 数据 |
| INT | 中断（可选） |
| AD0/CS | 地址选择（可选） |

用于后续接入 MPU6050 / ICM-42688 等 IMU 模块。

### 5.2 I2C IMU2 接口

与 IMU1 相同的 4~6Pin 排针接口。用于后续双传感器对比测试或冗余传感器配置。

### 5.3 PWM 舵机接口

至少 2 路舵机接口，每路包含：

| 引脚 | 信号 |
|------|------|
| PWM | STM32 定时器 PWM 输出 |
| 5V | 舵机电源 |
| GND | 共地 |

用于后续 SG90 舵机或执行器响应实验。舵机 5V 独立供电，不直接从 STM32 3.3V 取电。

### 5.4 RS485 预留接口

预留排针/插座，后续可配 MAX3485 或类似 RS485 收发器：

| 引脚 | 信号 |
|------|------|
| UART_TX | STM32 TX |
| UART_RX | STM32 RX |
| DE | 发送使能 |
| RE | 接收使能 |
| A / B | RS485 差分信号（接外部收发器模块） |

### 5.5 CAN 预留接口

先预留排针或模块插座位置，不强制本版完全启用。如果 STM32F103 的 CAN 引脚（PB8/PB9 被 OLED 占用，PA11/PA12 可用）有空闲，优先引出。

## 6. V2 PCB 暂缓模块

以下模块不建议在 V2 主线实现：

| 暂缓模块 | 原因 |
|----------|------|
| RK3588 / Linux 主控硬件 | 不属于当前 STM32+ESP32 HIL 测试板范围 |
| ESP32-S3 裸模组直接上板 | 当前 ESP32 开发板已满足需求，裸模组增加 PCB 设计复杂度和调试难度 |
| MicroSD 卡槽 | 暂无大容量存储需求 |
| RTC 时钟芯片 | STM32 内部 RTC 暂时够用 |
| GPS 模块接口 | 尚无明确验证场景 |
| 无刷电调 ESC 接口 | 当前只有直流有刷电机，无刷方案未验证 |
| 完整飞控电源系统 | V2 先保持简单的 5V → 3.3V 供电 |
| 四层板 | V2 两层板即可，信号速率低 |
| 高速 PCIe / PXIe / OcuLink | 不属于 STM32 测试板范围 |

## 7. 推荐板卡分区

```
┌─────────────────────────────────────────────┐
│  电源区 (5V in / 3.3V / GND Test Points)     │
├──────────┬──────────────────────────────────┤
│  STM32   │  ESP32-S3 开发板插座区             │
│  控制器区 │  (2×排母, 仅接 GND+通信信号)      │
│ (2×20    │                                    │
│  排母)    │                                   │
├──────────┴──────────────────────────────────┤
│  HIL 注入区                                  │
│  ESP32 GPIO4 → 1k → Jumper → STM32 PB14     │
├─────────────────────────────────────────────┤
│  传感器扩展区                                │
│  IMU1 I2C (4-6Pin)  /  IMU2 I2C (4-6Pin)   │
├─────────────────────────────────────────────┤
│  执行器 PWM 区                               │
│  PWM1 / PWM2 舵机接口 (3Pin each)            │

│  RS485预留 / CAN预留 / USART3 Debug (4Pin)   │

├─────────────────────────────────────────────┤
│  SWD 下载接口 / BOOT0 / RESET                │
└─────────────────────────────────────────────┘
```

## 8. V2 PCB 成功标准

V2 PCB 打样回来后，按以下顺序逐项验证：

- [ ] 板子上电无短路（万用表二极管档测 3.3V/GND、5V/GND）
- [ ] 3.3V 电源正常（万用表测 3.3V 测试点）
- [ ] 5V 电源正常（如使用 5V 输入）
- [ ] STM32 可通过 SWD 正常下载程序并运行
- [ ] ESP32 与 STM32 共地正常
- [ ] ESP32 GPIO4 可通过板上走线注入 PB14（ons 命令计数 +1）
- [ ] once 命令只触发一次计数
- [ ] auto 模式能根据 STATE 自动启停脉冲
- [ ] USART3 状态反馈可被 ESP32 读取（Serial Monitor + 网页显示正常）
- [ ] 逻辑分析仪能稳定夹取测试排针上的信号
- [ ] I2C 接口能接入并读取 MPU6050 或类似模块
- [ ] PWM 接口能驱动 SG90 舵机
- [ ] 跳线三态（真实传感器 / ESP32 / 断开）均正常工作

## 9. 当前结论

当前 STM32 + ESP32 HIL 雏形的接线已经过飞线验证：

- GPIO4 → PB14 虚拟传感器脉冲注入
- USART3 ↔ UART1 双向通信
- auto 模式状态闭环

日志记录完成、接口清单冻结后，可以开始设计 V2 PCB。

V2 不应做成完整飞控板，而应做成：

面向 HIL 半实物仿真、飞控控制器测试和硬件接口联调的**低成本控制器测试载板**。

设计完成后，V2 PCB 可同时用于：

- 个人 STM32 HIL 实验
- 公司飞控控制器测试方向的概念验证
- 后续 IMU / 舵机 / RS485 / CAN 接口的逐步扩展
