# 调试问题记录

## 1. PC13 按键无响应 (2026-06-04)

**现象**：SW2/PC13 按键按下后无任何反应，ADD 按键 (PB1) 正常工作。

**排查**：万用表测量 PC13 引脚电平，发现悬空/异常。

**根因**：PC13 引脚虚焊，STM32 无法读取正确的按键电平。

**解决**：补焊 PC13 引脚后恢复正常。

**教训**：PCB 焊接完成后应先做 GPIO 电平检查（所有按键引脚在未按时应为高电平 ~3.3V）。

---

## 2. 红外计数无反应 — PB14 电平恒为 1 (2026-06-08)

**现象**：
- 红外传感器本体 OUT 端电压可正常在 ~0V/~3V 之间变化
- 但 OLED 上 current_count 始终为 0
- 串口周期性日志 `PB14 lvl=1` 始终为 1，遮挡/未遮挡无变化

**排查过程**：

| 测量点 | 未遮挡 | 遮挡 | 判断 |
|--------|--------|------|------|
| 传感器 OUT (H4 输入端) | ~0V | ~3V | ✅ 传感器正常 |
| R1 靠传感器侧 | ~0V | ~3V | ✅ 信号过 R1 前正常 |
| R1 靠 PB14 侧 (IR 测试点) | ~1.6V | ~3V | ❌ 低电平被抬高 |

**根因**：R2 上拉电阻与 R1 串联分压，将 PB14 低电平从 ~0V 抬至 ~1.6V。

STM32F103 GPIO 输入阈值 (VDD=3.3V)：
- V_IL(max) = 0.35 × 3.3V = **1.155V** — 低于此值保证判为 0
- V_IH(min) = 0.65 × 3.3V = **2.145V** — 高于此值保证判为 1
- 1.6V 处于不确定区域 (1.155V ~ 2.145V)，STM32 实际判为 1

**临时修复**：R2 暂时空焊（不装），保留 R1 + C6。

修复后测量：
- PB14 未遮挡 ≈ 0V → `PB14 lvl=0` ✅
- PB14 遮挡 ≈ 3V → `PB14 lvl=1` ✅
- 上升沿触发正常，OLED 计数正常

**后续**：等 10K 电阻到货后评估是否补焊 R2；若补焊需重新测量 PB14 低电平是否 < 1.155V。

## 3. 电机接口 U2 焊盘损伤 (2026-06-04)

**现象**：电机输出接口 U2 焊盘受损。

**根因**：未安装 KF301 接线端子，直接焊线在焊盘上，焊接时过热/拉扯导致焊盘翘起。

**临时方案**：
- 暂停 U2 焊接操作，避免扩大损伤
- 等待 KF301-5.0-2P 接线端子到货
- 使用第二片 PCB 作为正式焊接板

**V1.1 改进**：U1/U2 接线端子周围预留更大焊盘间距，丝印标注焊接注意事项。

## 4. TB6612 发热异常 (2026-06-04)

**现象**：电机驱动供电测试中 TB6612 芯片异常发热。

**可能原因**：
- 电机供电接线不规范（杜邦线过细、接触不良）
- VM 电源未加滤波电容
- 电机堵转或负载过大

**处理**：暂停电机测试，等待接线端子和规范供电接线后再验证。

---

## 5. 软件 GetEvent/Update 调用顺序 bug (已修复)

**现象**：按键事件偶尔丢失，CONFIRM 响应不稳定。

**根因**：`App_Task()` 中 `App_Key_GetEvent()` 在 `App_Key_Update()` 之前调用，导致本帧产生的事件要到下一帧才能被消费。

**修复**：调整为 `App_Key_Update()` → `App_Key_GetEvent()` 顺序。

**注意**：Keil 使用 C89 模式，变量声明必须在函数块开头。修复时需将 `AppKeyEvent key` 等变量声明上移。

---

## 6. app_time.c 编译错误 (已修复)

**现象**：`error: #65: expected a ";"`

**根因**：`NVIC_InitTypeDef NVIC_InitStructure` 和 `sys_tick_ms = 0;` 被错误合并到同一行。

**修复**：拆分为两行独立语句。

---

## 调试经验总结

1. **上电前 GPIO 电平检查**：所有关键引脚 (按键、传感器输入) 在未触发时应为确定的高/低电平
2. **模拟信号 vs 数字阈值**：万用表测的电压值不等同于 STM32 读取的数字电平，需通过串口 `CountSensor_ReadRawLevel()` 确认
3. **分压电路谨慎**：输入端的 R1/R2 分压可能将信号推到不确定区域，焊接前应先计算分压值
4. **分阶段验证**：逻辑部分 → 传感器 → 电机驱动 → 全系统联合，每阶段确认后再推进
