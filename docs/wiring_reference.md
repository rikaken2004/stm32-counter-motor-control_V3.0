# 接线参考

## STM32 ↔ ESP32-S3 接线

| STM32F103 | ESP32-S3 | 信号方向 | 说明 |
|-----------|----------|----------|------|
| PB10 (USART3_TX) | GPIO18 (UART1 RX) | STM32 → ESP32 | 状态数据 |
| PB11 (USART3_RX) | GPIO17 (UART1 TX) | ESP32 → STM32 | RESET 命令 |
| GND | GND | — | **必须共地** |

## STM32 ↔ CH340 / Linux 接线

| STM32F103 | CH340 | 说明 |
|-----------|-------|------|
| PB10 (USART3_TX) | RXD | STM32 → Linux 数据 |
| GND | GND | 共地 |

只接收时不需要接 CH340 TXD。

## 供电原则

| 规则 | 说明 |
|------|------|
| 各自 USB 供电 | STM32 和 ESP32 各自用自己的 USB 线供电 |
| 必须共地 | 所有设备 GND 连在一起 |
| 不连接 3.3V | 两块板各自有 3.3V 稳压器，互连 3.3V 可能引起电源冲突 |
| 不接 5V 到 GPIO | 5V 会烧毁 ESP32/STM32 的 GPIO，GPIO 只支持 3.3V |

## ESP32-S3 引脚说明

| 引脚 | 本项目用途 | 备注 |
|------|-----------|------|
| GPIO18 | UART1 RX，接收 STM32 数据 | 独立 GPIO |
| GPIO17 | UART1 TX，向 STM32 发送命令 | 独立 GPIO |
| GPIO43 (U0TXD) | **不用** | 板载 CH340 下载串口 |
| GPIO44 (U0RXD) | **不用** | 板载 CH340 下载串口 |

GPIO43/44 已经连接给板载 CH340（用于烧录和 Serial Monitor），如果再用它们和 STM32 通信，会和烧录信号冲突。

## 面包板注意事项

- 面包板**电源轨道可能中间断开**（长面包板每隔一段有断口）。如果 STM32 和 ESP32 放在同一面包板上，需确认 GND 轨道连通
- 如果电源轨断开，需用跳线把断开的两段连起来
- 建议先把 GND 全连好，再接信号线，最后上电

## CH340 串口模块

- CH340 的 RXD 接 STM32 TX，TXD 接 STM32 RX
- CH340 只有一个 LED（通常是 RX 指示灯或电源灯）
- 如果 CH340 不工作，检查 USB 驱动或更换模块

## 快速检查清单

接线完成后，上电前过一遍：

- [ ] 所有 GND 互通（STM32 ↔ ESP32 ↔ CH340 ↔ 面包板）
- [ ] STM32 PB10 → ESP32 GPIO18（非接反）
- [ ] ESP32 GPIO17 → STM32 PB11（非接反）
- [ ] 未连接 3.3V 线（两块板各用 USB 供电）
- [ ] 未连接 5V 到任何 GPIO
- [ ] ESP32 未使用 GPIO43/44
- [ ] 波特率统一为 115200
