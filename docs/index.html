STM32 智能计数与电机控制系统

51 双 MCU 计数控制项目的 STM32 工程化升级版

本项目基于原 STC89C52 双 MCU 计数控制系统，迁移至 STM32F103C8T6 单 MCU 架构， 实现按键目标设定、红外计数、OLED 状态显示、电机 PWM 控制、达标停机、自动重启 和 USART3 调试日志。当前正在推进 PCB V1 模块化载板设计。

项目亮点

⚙

51 双 MCU → STM32 单 MCU

从两片 STC89C52 主从通信架构迁移至一片 STM32F103C8T6，精简硬件、统一软件栈

⏰

TIM4 1ms 非阻塞状态机

阻塞式 Delay_ms 替换为硬件定时器驱动的事件循环，零忙等，全状态响应

☱

PB14 EXTI 红外计数

外部中断下降沿捕获，delta 差值法消除累积误差，app 层 120ms 防抖过滤

☷

TIM3 PWM 电机控制

PA6 输出 PWM，PB12/PB13 方向控制，TB6612FNG H 桥驱动，速度可调

▤

OLED 实时状态显示

4 行信息：目标值 / 当前计数 / 累计 / 状态&电机，100ms 事件驱动刷新

✎

USART3 事件日志

PB10/PB11 调试串口，115200bps，按键/状态转换/计数事件实时输出

⚠

电机干扰定位

确认电机 EMI 对 PB14 红外信号的干扰路径，制定独立供电 + RC 滤波整改方案

★

PCB V1 模块化载板

从杜邦线搭棚升级为核心板 + 模块载板，嘉立创 EDA 设计，预留抗干扰措施

已定位问题：电机干扰

⚠ 现象

接入电机电源后，PB14 红外输入出现异常大脉冲（IR raw=25、 IR raw=30），可能导致计数异常或 MCU 复位。
断开电机电源后，仅保留 STM32、OLED、红外、按键、USB-TTL，红外计数恢复正常（raw=1）。

当前状态

软件状态机、按键、OLED、UART、红外基础计数逻辑 — 全部验证正常。
问题根因已定位为电机供电扰动和 EMI 对 PB14 红外信号线的传导/辐射干扰。App 层已加入 120ms 计数防抖作为软件兜底。

后续硬件整改

⚡ 独立供电

电机使用独立 5V~6V 电源，不与 STM32 共用

🌐 共地

STM32 GND ↔ 电机驱动 GND ↔ 电机电源 GND

🔌 电解电容

VM/GND 并联 100μF~470μF，吸收电流尖峰

🔌 陶瓷电容

VM/GND 并联 0.1μF；电机两端并联 0.1μF

🔗 RC 滤波

红外 OUT → 1kΩ → PB14 + 100nF 对地

📏 信号隔离

红外信号线远离电机 PWM/电源走线，PB14 预留 10kΩ 上拉
